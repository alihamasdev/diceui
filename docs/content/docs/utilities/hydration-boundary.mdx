# Hydration Boundary

The Hydration Boundary component helps manage hydration boundaries in React Server Components, preventing hydration mismatches and optimizing client-side hydration.

## Features

- Prevents hydration mismatches
- Optimizes client-side hydration
- Works seamlessly with React Server Components
- Zero dependencies

## Installation

```bash
npm install @diceui/hydration-boundary
```

## Usage

```tsx
import { HydrationBoundary } from "@diceui/hydration-boundary"

export default function App() {
  return (
    <HydrationBoundary fallback={<LoadingSpinner />}>
      <ClientComponent />
    </HydrationBoundary>
  )
}
```

## API Reference

### HydrationBoundary

The main component that creates a hydration boundary.

#### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `fallback` | `ReactNode` | - | Component to show while hydrating |
| `children` | `ReactNode` | - | The components to be hydrated |
| `suspense` | `boolean` | `true` | Whether to use Suspense boundary |

### useHydrated

A hook to check if a component is hydrated.

```tsx
import { useHydrated } from "@diceui/hydration-boundary"

function Component() {
  const isHydrated = useHydrated()
  
  return isHydrated ? <ClientOnlyComponent /> : null
}
```
